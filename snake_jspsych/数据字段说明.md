## 1. 食物相关字段

### 1.1 `food_locations`

**含义：** 目标食物（苹果）在网格中的位置编号（1-based 索引）

**数据格式：** 数组，如 `[3, 17]`

**解读：**
- 数字代表食物在 `all_locs_xy` 网格中的位置编号
- 编号从 1 开始
- 数组长度 = 该试次的目标食物数量

**示例：**
```
food_locations: [3, 17]
# 表示本试次有2个目标食物，分别在网格位置3和17
```

---

### 1.2 `food_order`

**含义：** 被试实际吃到苹果的顺序（网格位置编号）

**数据格式：** 数组，如 `[17, 3]`

**解读：**
- 记录被试**实际**吃到苹果的顺序
- 数字是网格位置编号（与 `food_locations` 对应）
- 可用于分析被试的路径选择策略

**示例：**
```
food_locations: [3, 17]    # 苹果在位置3和17
food_order: [17, 3]        # 被试先吃了位置17的苹果，再吃位置3的
```

---

### 1.3 `initial_optimal_order`

**含义：** 试次开始时，从蛇头位置到各苹果的**最优收集顺序**（按距离排序）

**数据格式：** 数组，如 `[0, 1]` 或 `[1, 0, 2, 3]`

**解读：**
- 数字是苹果的**索引**（0-based），对应 `food_locs_xy` 中的顺序
- `[0, 1]` 表示苹果0比苹果1更近，最优策略是先吃苹果0
- `[1, 0, 2, 3]` 表示苹果1最近，然后是0、2、3

**用途：**
- 对比 `food_order` 和 `initial_optimal_order`，分析被试是否按最优路径行动
- 评估空间规划能力

**示例：**
```
food_locs_xy: [(400, 120), (680, 240)]   # 苹果0在(400,120)，苹果1在(680,240)
initial_optimal_order: [0, 1]             # 苹果0更近，最优顺序是先吃0
food_order: [1, 2]                        # 实际先吃了位置1的（对应苹果1）
# 结论：被试没有按最优路径，可能存在空间记忆偏差
```

## 2. 提醒功能字段

### 2.1 `reminder_presses_before_food`

**含义：** 在吃到**每个**目标食物之前，使用提醒功能（F键）的次数

**数据格式：** 数组，如 `[0, 1]` 或 `[2, 0, 1]`

**解读：**
- 数组长度 = 目标食物数量
- 每个数字表示吃到对应苹果**之前**按了几次 F 键
- `[0, 1]` 表示：吃第1个苹果前没按F键，吃第2个苹果前按了1次
```
Trial 1: reminder_presses_before_food = [0, 0]     # 没用提示
Trial 2: reminder_presses_before_food = [1, 0]     # 吃第1个苹果前用了1次提示
Trial 3: reminder_presses_before_food = [0, 2, 1]  # 吃第2个前用2次，吃第3个前用1次
```

---

### 2.2 `reminder_onset` 和 `reminder_offset`

**含义：** 
- `reminder_onset`: 按下 F 键的时间点（毫秒）
- `reminder_offset`: 提示结束的时间点（毫秒）

**数据格式：** 数组，如 `[15000, 28000]`

**用途：**
- 精确分析提示使用的时机
- 计算提示持续时间

---

## 3. 蛇移动轨迹字段

### 3.1 `snake_pos`

**含义：** 试次过程中蛇头位置的时间序列（像素坐标）

**数据格式：** 坐标数组，如 `[(960, 440), (1000, 440), (1040, 440), ...]`

**解读：**
- **是的，这就是蛇移动的完整轨迹**
- 每个坐标 `(x, y)` 是蛇头在某一帧的位置
- 坐标单位是像素
- 配合 `time` 字段可以还原蛇的完整移动过程

**用途：**
- 绘制蛇的移动路径图
- 分析移动效率和策略
- 计算总移动距离

**示例：**
```
snake_pos: [(960, 440), (1000, 440), (1040, 440), (1040, 480)]
# 蛇从(960,440)开始，向右移动到(1000,440)，再到(1040,440)，然后向下到(1040,480)
```

## 字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `food_locations` | 数组 | 目标食物的网格位置编号 |
| `food_order` | 数组 | 实际吃到苹果的顺序（网格编号） |
| `initial_optimal_order` | 数组 | 初始最优收集顺序（苹果索引） |
| `updated_optimal_order` | 数组 | 吃完葡萄后更新的最优顺序 |
| `reminder_presses_before_food` | 数组 | 吃每个苹果前的提示次数 |
| `reminder_onset` | 数组 | 按F键的时间点 |
| `snake_pos` | 坐标数组 | 蛇头移动轨迹 |
| `time` | 数组 | 对应snake_pos的时间戳 |
| `eaten_type` | 数组 | 吃到的水果类型顺序 |
| `eaten_position` | 坐标数组 | 吃到的水果位置顺序 |
| `eaten_time` | 数组 | 吃到水果的时间顺序 |
